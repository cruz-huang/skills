# 投标文件检查规则（v4.0，legacy）

> 本文件为 v2.x 历史规则归档，只作追溯参考，不作为 `bid-document-checker v3.0.0` 当前执行依据。

## 核心原则

**"零废标、拿满分、控全周期风险"**

以招标文件为唯一标尺：
- 先定生死门槛
- 再拆得分规则
- 后挖隐性要求

---

# 一、前置准入与废标风险规则（生死线）

## 1.1 投标人资格门槛核查

### 主体资格
```
检查点：
- [ ] 法人主体要求（是否接受联合体）
- [ ] 是否允许分支机构投标
- [ ] 注册地/属地化要求
- [ ] 营业执照经营范围与项目匹配
- [ ] 独立法人资格
```

### 硬性资质
```
检查点：
- [ ] 系统集成资质
- [ ] CMMI认证
- [ ] 建筑施工资质
- [ ] 安全生产许可证
- [ ] 等级完全满足
- [ ] 在有效期内
- [ ] 认证范围匹配
```

### 人员门槛
```
检查点：
- [ ] 注册建造师证
- [ ] 安全生产考核证
- [ ] PMP证书
- [ ] 职称要求
- [ ] 社保缴纳要求
- [ ] 无在建项目承诺
```

### 禁止投标情形
```
检查点：
- [ ] 失信被执行人
- [ ] 重大违法记录
- [ ] 关联单位禁入
- [ ] 母公司与子公司不同时投标
```

## 1.2 实质性条款与废标条款

### 关键词检索
```
★、实质性要求、不满足即废标、否则作废标处理、
必须提供、不得偏离、禁止、不接受
```

### 核心刚性条款
| 条款 | 最低要求 |
|------|---------|
| 投标有效期 | ≥ 招标要求时长 |
| 投标保证金 | 金额/方式/到账时间/凭证合规 |
| 工期/交付期 | ≤ 招标要求 |
| 质保期 | ≥ 招标要求 |
| 质量标准 | 完全响应 |
| 付款方式 | 不得修改核心条款 |
| 不转包不分包 | 完全响应 |

## 1.3 程序性红线
```
检查点：
- [ ] 投标截止时间
- [ ] 开标地点
- [ ] 文件递交方式
- [ ] 密封要求
- [ ] 份数要求
- [ ] 格式要求
```

## 1.4 招标固定格式检查

招标文件给出固定格式时，固定格式本身是质量门：

```
检查点：
- [ ] 投标书/投标函按招标模板字段、顺序、下划线、签章和日期位置编制
- [ ] 法定代表人资格证明书按招标模板编制
- [ ] 授权委托书按招标模板编制
- [ ] 承诺函按招标模板编制
- [ ] 偏离表、报价表、人员表、业绩表按招标模板保留表头和字段
- [ ] 缺资料时只在对应字段保留“待定”或空白占位；提交前不得残留
- [ ] 未见固定格式时，报告标注“未见招标固定格式模板”
```

固定格式材料被改写为自由文本的，即使语义看似完整，也应判为返工。

# 二、商务评分规则检查

## 2.1 评分权重
```
检查点：
- [ ] 商务分占比
- [ ] 资格项 vs 评分项区分
- [ ] 评分方式（区间/阶梯/符合即满分）
- [ ] 缺项扣分规则
```

## 2.2 得分项核查

### 企业综合实力
| 得分项 | 证明材料 |
|--------|---------|
| ISO认证 | 证书原件、认证范围匹配 |
| 知识产权 | 专利/软著证书 |
| 企业荣誉 | 获奖证明、时间范围符合 |
| 财务状况 | 审计报告、无连续亏损 |

### 业绩（高频扣分）
```
检查点：
- [ ] 同类项目定义匹配
- [ ] 业绩数量满足
- [ ] 单项金额≥要求
- [ ] 时间范围（近X年）
- [ ] 证明材料完整
```

### 团队配置（高频扣分）
```
检查点：
- [ ] 项目经理资质
- [ ] 技术负责人资质
- [ ] 团队成员配置
- [ ] 社保缴纳证明
- [ ] 人员到岗要求
```

## 2.3 商务偏离表
```
检查点：
- [ ] 顺序与招标一致
- [ ] 无漏项
- [ ] 无负偏离
- [ ] 正偏离有标注
- [ ] 保留招标原文符号、条款、参数和关键条件
- [ ] 不精简、不合并、不改写招标关键表述
```

# 三、技术评分规则检查

## 3.1 技术需求分层

| 层级 | 响应要求 |
|------|---------|
| ★实质性条款 | 100%响应，有参数/方案/佐证 |
| 一般性需求 | 逐条响应最低标准 |
| 核心痛点需求 | 国产化/信创/等保/数据安全 |

## 3.2 技术评分项

| 评分项 | 评审标准 |
|--------|---------|
| 方案架构 | 符合国家行业标准、架构清晰 |
| 重难点分析 | 识别精准、有针对性、可落地 |
| 实施能力 | 进度管控、现场管理、安全文明 |
| 适配对接 | 对接方案、原厂承诺 |

## 3.3 技术方案完整性
```
检查点：
- [ ] 需求分析
- [ ] 系统设计
- [ ] 开发计划
- [ ] 测试方案
- [ ] 部署上线
- [ ] 运维保障
- [ ] 功能点100%覆盖
```

# 四、报价合规检查

## 4.1 报价格式
```
检查点：
- [ ] 按招标模板格式
- [ ] 无漏项/多报
- [ ] 暂列金按给定金额
- [ ] 暂估价按给定金额
```

## 4.2 算术准确性
```
检查点：
- [ ] 单价×数量=合价
- [ ] 分项之和=总价
- [ ] 大小写一致
- [ ] 全文件统一
- [ ] 报价总价、税额、维护费、税率、分项报价只从确认来源同步
```

## 4.3 报价合规
```
检查点：
- [ ] 税金按国家规定
- [ ] 规费按地方规定
- [ ] 无低于成本价
- [ ] 无不平衡报价
- [ ] 小微企业声明（如适用）
- [ ] 节能环保证明（如适用）
```

## 4.4 高风险字段唯一出处

```
检查点：
- [ ] 报价总价、大小写金额、税额、税率、维护费一致
- [ ] 分项报价与报价汇总表一致
- [ ] 技术 BOM、设备数量、关键配置与报价/技术响应/配置表一致
- [ ] 交货期、质保期、服务期限与投标函、承诺函、商务响应一致
- [ ] 多处出现的高风险字段能追溯到同一确认口径
```

无法从投标文件或主控确认口径判断唯一出处时，报告写“需用户确认来源”，不得擅自选定口径。

```

- [ ] 权责划分不对等
- [ ] 知识产权归属
- [ ] 保密范围/期限
```

## 5.3 变更风险
```
检查点：

- [ ] 价格波动风险

```

# 八、错别字与文字准确性检查

## 核心原则

**不限定范围，自由发现文字异常**

## 8.1 发现的问题类型

| 问题类型 | 示例 | 说明 |
|---------|------|------|
| Word修订残留 | "对 于"（应为"对于"） | 最严重的复制粘贴错误 |
| 拼音标注残留 | "里 程 碑"（应为"里程碑"） | Word拼音功能残留 |
| 双句号 | "排序。。" | 标点错误 |
| 跨行业术语 | 水务项目出现"海关" | 复制粘贴错误 |
| 文字被分开 | "日 期" | PDF提取断行导致 |

## 8.2 检查方法

```
1. 逐页扫描全部文字（不预设关键词）
2. 识别不符合正常书写习惯的模式
3. 特别关注：
   - 被空格分开的汉字（"对 于"）
   - 连续的标点符号（"。。"、"，，"）
   - 上下文语义不通顺
   - 不符合项目行业/地区的术语
4. 验证发现的异常是否为真实错误
```

## 8.3 Word文档特有错误

Word转PDF时可能残留的问题：
- 修订模式文字（显示删除线但未实际删除）
- 拼音标注（文字上方有拼音）
- 批注内容
- 文本框内容可能被截断

## 8.4 内部编制信息泄漏

正式投标正文、送审版和最终 PDF 中不得出现：
- `资料引用`
- `检查点`
- `未闭环事项`
- `待确认`
- `TODO`
- 面向内部写作/复核的注释块

如出现，应列为提交前阻断项或至少高风险问题。

## 8.5 交付物阶段和命名检查

检查文件路径和文件名是否体现阶段和版本：
- `02-工作稿` 为内部制作区，不应作为对外交付物路径。
- `03-交付物` 为可流转成果区。
- 最终版只能从检查通过的送审版或盖章版产生，不应由过程稿直接改名。
- 文件名应体现送审版、盖章版、最终版、版本号或日期等阶段信息。

## 8.6 输出格式

```json
{
  "typo_check": {
    "total_pages": 97,
    "issues": [
      {
        "page": 58,
        "content": "对 于项目实施主计划",
        "correct": "对于项目实施主计划",
        "severity": "critical",
        "type": "word_revision_residue"
      }
    ]
  }
}
```

# 执行流程

## 第一步：快速通读（1小时）
```
产出：可行性判断
```

## 第二步：逐字精读+三色标注
```
红色 = 废标条款
黄色 = 评分项
蓝色 = 格式/履约风险

产出：
1. 《废标条款负面清单》
2. 《得分项响应责任清单》
3. 《投标执行动作清单》
```

## 第三步：交叉复核
```
复核：遗漏/矛盾/模糊/违法条款
产出：答疑清单
```

## 第四步：最终闭环
```
责任到人，时间到点
```

# 评分标准

| 分数 | 等级 |
|------|------|
| 90-100 | 优秀 |
| 80-89 | 良好 |
| 70-79 | 合格 |
| 60-69 | 风险 |
| <60 | 危险 |
