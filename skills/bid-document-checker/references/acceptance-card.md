# 投标文件检查验收卡

版本：v3.0.4  
用途：`bid-document-checker` 的内置通用验收卡模板。

本卡是投标文件正式验收标准的唯一来源。

结构化中间文件（`document_blueprint.json`、`*_blocks.json`、字段 JSON、模板路径）由 `bid-reviewer` 做过程复核；本卡不逐个验收中间 JSON，只检查最终 DOCX/PDF/交付稿是否正确呈现。最终文件残留占位符、缺图、漏表、空表、固定格式未渲染或内部块外露，仍按正式验收问题处理。

## 1. 使用规则

验收卡是一套体系，不再分成“检查域”和“专项视角”两套清单。  
本文件是全局模板，不是项目要求全集；每次检查必须先形成“本轮检查项”。`formal_report` 将本轮检查项落盘为“本轮整合验收卡”项目实例卡，`lightweight_check` 只在结构化摘要中列明覆盖情况，不落盘验收卡文件。

输出模式：

- `lightweight_check`：轻量检查，只输出结构化摘要，不生成 Markdown、HTML 或验收卡文件；仍必须使用本验收卡标准判断。
- `formal_report`：正式报告，生成本轮整合验收卡、Markdown 报告和 HTML 用户版；向用户交付时优先提供 HTML 用户版。

最小结构：

- `PRE`：前置检查。
- `G01-G06`：通用检查，负责横向底线。
- `D00-D11`：内容域检查，负责投标大纲和投标文件具体模块。

`formal_report` 本轮整合验收卡字段固定如下；`lightweight_check` 摘要可压缩呈现，但必须保留检查项覆盖、P0/P1/P2 和返工路径：

| 状态 | 检查ID | 检查项 | 招标或项目依据 | 检查动作 | 结论（位置/结果/风险/问题/建议） |
|---|---|---|---|---|---|

状态只允许：`通过`、`P0`、`P1`、`P2`、`不适用`、`受限未执行`。  
`当前问题` 清单是 P0/P1/P2 的唯一完整问题台账，P0/P1/P2 数量只按清单中的唯一编号统计。`结论（位置/结果/风险/问题/建议）` 合并填写：通过项只写 `通过`；问题项写 `证据位置；关联 P1-01；详见当前问题清单。`；不适用或受限项写清原因。

逐项核对规则：

- 必须先形成本轮检查项，再检查，再回填。
- `formal_report` 必须将本轮检查项落盘为本轮整合验收卡；`lightweight_check` 只在回报中列明检查项覆盖摘要，不落盘文件。
- 不得批量写“均通过”“全部符合”代替逐项核对。
- 通过项可只写 `通过`；P0/P1/P2 行必须写明证据位置并关联唯一问题编号，不得重复展开完整问题、风险和建议。
- 每个 P0/P1/P2 问题必须能反查到至少一个检查 ID。
- 每个 P0/P1/P2 问题必须给出最小返工路径：问题源头、责任技能、是否需要改 MD、是否需要改资料清单/证据矩阵、是否需要重排 Word、复查范围。
- 每个 P0/P1/P2 问题的完整描述、风险和整改建议只写在 `当前问题` 清单；G/D/PRE 明细只写 `证据位置；关联 P1-01；详见当前问题清单。`
- `未回填项数` 不为 0 时，不得输出完整检查结论。

覆盖率摘要：

| 总检查项数 | 项目增量项数 | TENDER项数 | 已完成项数 | 不适用项数 | 受限未执行项数 | 未回填项数 | 未映射招标要求数 | 是否允许完整结论 |
|---:|---:|---:|---:|---:|---:|---:|---:|---|

## 2. 动态生成规则

1. 先读招标文件、补遗答疑、招标解析产物；有项目口径时同步读取。
2. 再读取本验收卡模板，形成本轮检查项。
3. 将 `requirements`、`fatal_checklist`、`scoring_matrix`、`material_checklist`、`fixed_formats`、`timeline_matrix` 映射到 `G/D` 检查项。
4. 大纲、目录、分册、投标文件格式、章节结构要求优先映射到 `D00`；涉及固定格式名称和签章位置时同步关联 `G04`。
5. 每条有效招标要求至少映射到一个主责检查项；可有多个关联检查项，但问题只编号一次。
6. 能归入 `G/D` 的不新增项；确实找不到合适位置时才新增 `TENDER-xx`，不再新建第二套专项矩阵。
7. 未映射的资格项、废标项、评分项、证明材料项、固定格式项、目录/大纲项或时间周期项必须进入 `TENDER-xx`、人工复核或受限说明；未处理不得输出完整通过。
8. 通用检查不能反向创造招标未要求的材料；但投标文件主动提供的材料必须检查有效期、主体、清晰度和一致性。

`formal_report` 的本轮整合验收卡输出位置：

- 有项目目录：`04-检查反馈/投标文件检查验收卡-{{检查阶段}}-{{版本}}.md`
- 临时检查：`<投标文件目录>/.bid-checker/投标文件检查验收卡.md`

`lightweight_check` 不生成上述文件，必须在结构化摘要中写明总检查项、TENDER项、受限项、未映射项、P0/P1/P2 和最小返工路径。

招标要求映射摘要建议格式：

| 来源 | 有效要求数 | 已映射 | TENDER项 | 人工复核/受限 | 未映射 |
|---|---:|---:|---:|---:|---:|
| requirements | {{数量}} | {{数量}} | {{数量}} | {{数量}} | {{数量}} |
| fatal_checklist | {{数量}} | {{数量}} | {{数量}} | {{数量}} | {{数量}} |
| scoring_matrix | {{数量}} | {{数量}} | {{数量}} | {{数量}} | {{数量}} |
| material_checklist | {{数量}} | {{数量}} | {{数量}} | {{数量}} | {{数量}} |

## 3. PRE 前置检查

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| PRE-01 | 检查阶段已确认 | 记录章节版本、初稿版本、定稿版本或盖章版本；无法判断时先问用户。 |
| PRE-02 | 检查轮次已确认 | 记录 first_check、recheck、spot_check 或 final_gate；复查需读取上一轮问题。 |
| PRE-03 | 检查对象和版本已确认 | 记录文件名、路径、页数、hash 或版本号；避免把旧版/过程稿当当前对象。 |
| PRE-04 | 招标解析质量门已读取 | 核对 parse_manifest、source hash、quality_gate、manual_review_queue。 |
| PRE-05 | 上游成果已读取 | 有项目控制台时核对响应矩阵、资料清单、整改闭环；ad_hoc_mode 核对招标解析产物。 |
| PRE-06 | 验收卡来源已确认 | 写明本技能验收卡读取情况。 |
| PRE-07 | 视觉核验方式和边界已确认 | 记录 model_vision、recognizer_tool、hybrid 或 text_only_limited；CA/电子签章有效性、证书/信用截图外部真伪、递交平台/上传/加密状态属于边界外。 |
| PRE-08 | 输出信息已准备 | `formal_report` 的报告首页必须包含项目、招标人、投标人、招标文件、检查对象、页数、时间、阶段和结论；`lightweight_check` 摘要必须包含检查对象、阶段、结论、受限项和是否需用户确认。 |
| PRE-09 | 最小返工口径已确认 | 本轮报告的问题整改建议按源头分流，不默认全流程重跑；无法判断源头时标为需主控判定。 |

## 4. G01-G06 通用检查

通用检查只做横向兜底，不重复展开 D00-D11 的全部细项。

### G01 核心信息一致性

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G01-01 | 投标主体一致 | 核对封面、投标函、报价表、偏离表、营业执照、授权书、盖章主体。 |
| G01-02 | 项目名称、招标编号、招标人一致 | 核对封面、投标函、授权售后材料、承诺函、报价表、证明材料。 |
| G01-03 | 高风险字段全篇一致 | 核对报价、税率、数量、型号、BOM、交付期、质保期、付款、验收。 |
| G01-04 | 已提供材料不自相矛盾 | 检查附件材料与正文、偏离表、证明材料索引是否冲突。 |

### G02 证据可追溯性

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G02-01 | 响应和证明有最小定位 | 核对章节、页码、图号、附件标题、证明材料名是否可定位。 |
| G02-02 | 不编造页码和证明编号 | 检查页码、截图编号、证书编号、附件编号是否真实存在。 |
| G02-03 | 招标要求可映射到检查项 | 废标项、资格项、评分项、技术参数、商务条款、递交要求至少映射到一项检查卡。 |
| G02-04 | 问题可反查验收卡 | 当前问题清单中的每个 P0/P1/P2 问题绑定检查 ID、证据位置和整改建议；明细行只引用问题编号。 |
| G02-05 | 最小返工路径清楚 | 每个问题说明应回到资料、写作、方案、DOCX、盖章或主控哪个源头，并限定复查范围。 |

### G03 图片/扫描件可读性

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G03-01 | 图片类证明文件清晰可读 | 核验证书、合同、授权书、售后函、截图、身份证、社保、个税等关键字段可读。 |
| G03-02 | 图片方向和裁切正常 | 核验横置、倒置、裁切、变形、缩放过小等影响阅读的问题。 |
| G03-03 | 签章、签名、日期可见事实已核验 | 按视觉模式核验 PDF 内可见事实；无视觉时标为受限；不判断 CA 有效性和外部真伪。 |
| G03-04 | 图片证明与正文描述一致 | 核对证书等级、有效期、主体、型号、日期、项目名称与正文是否一致。 |

### G04 固定格式、签章、日期底线

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G04-01 | 招标固定格式已识别 | 从招标“投标文件格式”和解析产物核对，不只看目录命中。 |
| G04-02 | 固定格式未被破坏 | 核对标题、表头、字段顺序、固定原文、下划线、说明和注释。 |
| G04-03 | 签字位、盖章位、日期位正确 | 核对位置未移动、未空缺；需要手签的位置不得只打印姓名冒充签字；电子签章只核对 PDF 可见事实。 |
| G04-04 | 固定格式材料分页和视觉结构可读 | 核对签章区、表格、落款、身份证位置、日期位置不跨页变形。 |

### G05 页码编号、文字质量、内部痕迹

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G05-01 | 页码和目录连续一致 | 核对 PDF 物理页、页脚页码、目录页码、附件引用页码，不得断页、跳页、重号、倒序或缺失。 |
| G05-02 | 编号连续 | 检查大纲标题、方案分项、表格行号、参数序号、图号、表号、附件编号。 |
| G05-03 | 中文文本和标点质量 | 正式中文文本优先中文标点；固定格式原文、型号、网址、邮箱、证书原文不得误改。 |
| G05-04 | 内部痕迹清除 | 检索 BID_META、DOCX_FORMAT、TODO、待补、资料路径、内部说明等。 |
| G05-05 | 表格和分页可读 | 检查跨页表头、版心、边框、文字挤压、标题孤行、附件标题与图片对应。 |
| G05-06 | 结构化生成痕迹清除 | 检查最终 DOCX/PDF 中不得残留 `{{table: ...}}`、`{{image: ...}}`、`{{form: ...}}`、`{{appendix: ...}}`、`BID_META`、`DOCX_FORMAT`、`DOCX_HANDOFF` 或字段 JSON 内容；图片、表格、固定表单必须已正确渲染。 |

### G06 废标/否决项检查

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| G06-01 | 废标/否决条款已提取 | 从招标文件提取废标、否决投标、无效投标、不予受理、资格/符合性审查不通过、实质性不响应等条款，形成逐项清单。 |
| G06-02 | 资格性废标项核对 | 核对主体资格、资质证书、信用记录、税收社保财务证明、授权代表、联合体限制等资格性否决项。 |
| G06-03 | 符合性/商务废标项核对 | 核对投标函、报价、商务偏离、合同条款、投标有效期、保证金/保函、递交要求等符合性否决项。 |
| G06-04 | 技术实质性负偏离核对 | 核对关键技术条款、★/▲条款、证明材料、参数响应和技术偏离表是否触发实质性负偏离。 |
| G06-05 | 固定格式、签章、日期和递交红线核对 | 核对固定格式破坏、签字盖章日期缺失、法定代表人/授权委托、密封/加密/上传/递交要求等红线。 |
| G06-06 | P0 问题映射和清零状态 | 将触发废标/否决风险的问题映射到 P0；无 P0 时写明逐项核对未触发，存在 P0 时不得建议提交。 |

## 5. D00-D11 内容域检查

### D00 投标大纲

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| D00-01 | 招标大纲要求已识别 | 判断招标是否有分册、章节、目录、投标文件格式或固定格式标题要求；必须排除目录页同名误判。 |
| D00-02 | 强制大纲与招标一致 | 有强制要求时，核对标题、顺序、层级、分册、固定格式名称是否与招标一致。 |
| D00-03 | 自拟大纲覆盖完整 | 无强制大纲时，核对资格、报价、商务、技术、方案、证明、授权售后、承诺签章等入口完整。 |
| D00-04 | 评分项有章节入口 | 核对评分项、加分项、证明材料项是否能定位到明确章节、小节或附件入口。 |
| D00-05 | 下级标题来源合理 | 二级/三级标题只能来自招标条款、评分项、固定格式/资料项或方案必要构成，不为丰富而新增无关标题。 |
| D00-06 | 大纲与正文/目录一致 | 核对目录标题、正文标题、页码、附件索引、证明材料入口一致。 |
| D00-07 | 大纲不制造口径风险 | 核对服务期、工期、交货期、验收、质保、付款等标题不误导或冲突。 |

| D01-02 | 总价一致 | 核对投标函、开标一览表、总价表、报价汇总表。 |
| D01-03 | 大小写金额一致 | 逐处比对金额大小写和单位。 |
| D01-04 | 税率税额可复算 | 复算含税、不含税、税率、税额关系。 |
| D01-05 | 分项合计等于总价 | 复算数量、单价、合价、分项合计。 |
| D01-06 | 数量、型号、配置与技术响应一致 | 核对设备、软件、授权、服务、配件数量和 BOM。 |
| D01-07 | 报价范围覆盖招标要求 | 检查运输、保险、安装、调试、培训、文档、验收、质保、售后。 |
| D01-08 | 维护费或维保比例符合限制 | 核对比例、基数、金额和计算口径。 |

### D02 商务偏离表

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| D02-01 | 商务条款覆盖完整 | 覆盖报价、付款、交付、验收、质保、售后、合同责任、有效期、签章授权。 |
| D02-02 | 条目号来自招标原文 | 不使用响应矩阵编号、自建编号替代招标条款号。 |
| D02-03 | 响应栏先写明确结论 | 检查是否先写“我司完全响应。”或“完全响应，无偏离。”。 |
| D02-04 | 无实质性负偏离 | 对照 ★、实质性商务条款、不得偏离条款。 |
| D02-05 | 证明位置可追溯 | 指向最小章节、附件标题、图号、资料名或最终 PDF 页码。 |
| D02-06 | 高风险商务字段一致 | 报价、税率、维护费、付款比例、交付期、质保期不得冲突。 |

### D03 技术偏离表

| 检查ID | 检查项 | 检查动作 |
|---|---|---|
| D03-01 | 技术要求逐条覆盖 | 对照技术条款、重要技术条款响应表和响应矩阵。 |
| D03-02 | 重点条款覆盖 | ★、▲、废标项、评分项、证明材料条款不得漏项。 |
| D03-03 | 证明材料位置完整 | 每个要求提供证明的条款有截图、彩页、证书、检测报告、官网材料等位置。 |

| D03-06 | BOM/型号/数量/配置一致 | 核对报价清单、技术响应、产品证明、截图、彩页。 |
